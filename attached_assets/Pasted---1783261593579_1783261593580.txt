Ты уже построил сильную основу, но теперь нужно перейти от словаря и уроков к действительно полезному языковому помощнику для обычных пользователей.

## Главная цель
Сделать Andi Language Platform не только учебным сайтом, но и практическим инструментом:
- словарь,
- уроки,
- фразы,
- translation assistant,
- future audio pipeline,
- recording workflow для носителей.

## Важная установка
Не жди готового аудио как обязательного условия запуска.
Если открытых аудиозаписей мало или нет, честно фиксируй это и строй систему так, чтобы:
- текстовая часть работала уже сейчас,
- аудио можно было добавить позже,
- записи можно было собирать вручную у носителей.

## Что нужно сделать
### 1. Добавить translation assistant
Сделай режим, где пользователь может:
- ввести русское предложение,
- получить черновой перевод на Андийский,
- увидеть построчный/пофразовый разбор,
- увидеть уровень уверенности для каждого фрагмента,
- понять, какие куски точные, а какие требуют ручной правки.

Важно:
- это не должен быть фальшивый “полный машинный перевод”;
- если перевода нет, система должна честно сказать “нет уверенного соответствия”;
- для неизвестных конструкций показывай шаблонные подсказки и примеры из словаря.

### 2. Сделать phrasebank
Добавь отдельный банк фраз:
- приветствия,
- бытовые фразы,
- просьбы,
- вопросы,
- числа,
- время,
- семья,
- еда,
- дом,
- базовые действия.

Фразы должны быть связаны со словарём и уроками.
У каждой фразы должны быть:
- перевод,
- разбор,
- источник,
- confidence,
- пример использования.

### 3. Добавить lessons for practical use
Новые уроки должны быть не просто по грамматике, а по практике:
- “Как поздороваться”,
- “Как спросить имя”,
- “Как сказать, что тебе нужно”,
- “Как считать”,
- “Как описать дом и семью”,
- “Как задать вопрос”,
- “Как сказать время”.

Каждый урок:
- короткое объяснение,
- 3–5 фраз,
- 3–6 заданий,
- повторение,
- quick quiz,
- итог.

### 4. Сделать аудио-план без блокировки проекта
Если готового аудио-корпуса нет:
- не останавливай проект,
- создай структуру для будущих записей,
- добавь инструкции для записи носителями,
- добавь шаблон каталога аудио,
- добавь статус каждого аудио-элемента: missing / requested / recorded / verified.

Если возможно, добавь кнопку “Запросить запись” для будущего сбора у носителей.
Не выдумывай, что аудио есть, если его нет.

### 5. Подготовить план ручной записи
Создай файл с инструкциями для записи:
- список фраз и слов для дикторов,
- формат файлов,
- требования к качеству,
- имена файлов,
- метаданные,
- согласие носителя,
- как загружать в систему.

### 6. Улучшить словарь и фразы
Сделай акцент на:
- слова из реальной повседневной жизни,
- частотные слова,
- учебные фразы,
- глаголы действия,
- вопросительные слова,
- связки и служебные слова.

Если можно извлечь фразы из уже имеющихся материалов — делай это.
Если нельзя — создай backlog на дальнейшее наполнение.

### 7. Поддержать поиск и навигацию
Добавь поиск по:
- слову,
- фразе,
- теме,
- уроку,
- источнику,
- confidence,
- наличию audio status.

### 8. Переименовать продуктовые функции понятнее
Для обычного пользователя должно быть просто:
- Учить слова,
- Учить фразы,
- Практика,
- Переводчик,
- Прогресс.

Технические разделы оставь в advanced mode.

### 9. Честный отчёт
В конце обязательно покажи:
- сколько слов,
- сколько фраз,
- сколько уроков,
- есть ли аудио,
- что реально работает,
- что пока только в виде заготовки,
- какие данные нужны от человека вручную.

## Definition of done
Проект считается улучшенным только если:
- появился translation assistant,
- есть phrasebank,
- уроки стали практическими,
- аудио не является блокером,
- система честно показывает ограничения,
- обычный пользователь получает реальную пользу уже сейчас.
