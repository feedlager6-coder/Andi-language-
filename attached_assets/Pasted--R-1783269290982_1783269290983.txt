Ты уже построил сильную платформу. Теперь твоя задача — подготовить проект к стабильному деплою на Railway и сделать так, чтобы следующий агент или разработчик сразу понимал, как всё устроено и как безопасно запускать систему.

## Главная цель
Сделать проект:
- стабильно запускаемым,
- пригодным для Railway,
- документированным,
- понятным для будущих агентов,
- и не ломающимся при деплое.

## Что нужно сделать
### 1. Подготовить Railway deployment
Проверь и при необходимости исправь:
- package.json scripts (`dev`, `build`, `start`, `typecheck`, `seed`)
- переменные окружения
- подключение к PostgreSQL
- миграции/seed
- публичный домен
- healthcheck endpoint
- обработку ошибок при старте

Если проект full-stack, убедись, что frontend и backend запускаются корректно в production.

### 2. Сделать production-ready конфигурацию
Проверь:
- сборку без ошибок,
- запуск без dev-only зависимостей,
- корректные пути,
- отсутствие hardcoded localhost,
- корректные CORS/BASE_URL настройки,
- работу с Railway DATABASE_URL,
- корректную работу API endpoints в production.

### 3. Создать PROJECT_PLAYBOOK.md / AGENT_GUIDE.md
Создай файл, который будет главным навигатором по проекту для будущих Replit/Railway/CLI агентов.

Файл должен содержать:
- краткое описание проекта,
- архитектуру,
- структуру папок,
- что уже реализовано,
- как запускать локально,
- как запускать в production,
- какие переменные окружения нужны,
- как подключить PostgreSQL,
- как запускать миграции,
- как запускать seed,
- как проверять здоровье приложения,
- как добавлять слова, фразы и уроки,
- как работает переводчик,
- как работает grammarModel,
- что нельзя ломать,
- известные ограничения,
- что делать при деплое на Railway,
- типичные ошибки и их решения.

### 4. Сделать deployment checklist
Добавь отдельный чеклист:
- GitHub repo подключён.
- Railway service создан.
- PostgreSQL добавлен.
- ENV vars настроены.
- build/start scripts проверены.
- migrations run.
- seed run.
- domain generated.
- app opens in browser.
- API responds.
- dictionary loads.
- lessons load.
- translator works.
- phrasebank works.

### 5. Проверить runtime
Обязательно проверь после изменений:
- стартует ли приложение с нуля,
- нет ли ошибок на production build,
- все основные страницы открываются,
- БД подключается,
- данные читаются,
- нет ли упавших запросов.

### 6. Сделать честный отчёт
В конце ответь:
- готов ли проект к Railway,
- что нужно сделать вручную,
- есть ли неподдерживаемые шаги,
- какие env vars обязательны,
- что может сломаться при деплое.

## Формат результата
Сначала обнови код и конфиг.
Потом создай/обнови AGENT_GUIDE.md.
Потом выведи короткий production report.
Не пропускай проверку работоспособности.
