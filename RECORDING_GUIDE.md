# Руководство по записи аудио для андийского языка

## Зачем это нужно

В приложении сейчас **нет ни одной настоящей аудиозаписи**. Ни для слов, ни для фраз.
Мы принципиально не используем синтез речи (TTS) и не подделываем аудио — андийский язык
слишком мало документирован, и неверное произношение, выданное за настоящее, нанесёт
больше вреда, чем полное отсутствие звука. Вместо этого приложение честно показывает статус
записи для каждого слова и фразы:

| Статус | Значение |
|---|---|
| `missing` (нет аудио) | Записи нет вообще |
| `requested` (запись запрошена) | Пользователь нажал «Запросить запись» — попало в очередь |
| `recorded` (записано, не проверено) | Есть аудиофайл, но его ещё не подтвердил носитель/лингвист |
| `verified` (проверено носителем) | Носитель языка подтвердил корректность произношения |

## Кто может записывать

Нужен **носитель андийского языка** (гагатлинский говор — сс. Анди, Гагатль, Риквани, Зило)
или как минимум свободно владеющий говором человек, знакомый с его произношением.
Желательно 2+ человека для перекрёстной проверки — андийский язык не имеет единой
письменной нормы, и произношение может отличаться между сёлами.

## Как записывать (технические требования)

1. **Формат**: MP3 или WAV, 44.1 кГц, моно.
2. **Тишина**: без фонового шума (закрытая комната, без музыки/ТВ/ветра).
3. **Один файл — одно слово/фраза**: не записывайте список подряд одним файлом.
4. **Темп**: чуть медленнее обычной разговорной речи, отчётливая артикуляция, но без
   искусственной "дикторской" манеры.
5. **Повтор**: запишите каждое слово/фразу дважды — испытание в естественном темпе
   и медленно по слогам — если возможно.
6. **Имя файла**: используйте `word-<id>.mp3` для слов и `phrase-<id>.mp3` для фраз
   (id соответствует записи в базе данных приложения — можно посмотреть в админке
   или через API `/api/words/:id`, `/api/phrases/:id`).

## Что записывать в первую очередь

Приоритет — по статусу `requested` (запись запрошена пользователями) и по фразам
из практических уроков (приветствия, представление, вопросы, числа, семья). Их можно
увидеть во «Фразнике» приложения — там же есть кнопка «Запросить запись», которая
отмечает нужную фразу и помогает расставить приоритеты.

## Что делать с записанными файлами

1. Загрузить файлы в облачное хранилище проекта (Object Storage) — сообщите
   разработчику, он подключит хранилище и добавит загрузку в админку.
2. Обновить статус на `recorded` через админку или API
   (`PATCH /api/words/:id` или `/api/phrases/:id` с `{"audioStatus": "recorded"}`).
3. После проверки вторым носителем — обновить статус на `verified`.

## Важно

Никогда не помечайте аудио как `verified`, если его не подтвердил живой носитель языка.
Лучше честно показать пользователю «аудио пока нет», чем ввести его в заблуждение
неверным произношением.
